# Scope And Out Of Scope

## Alcance de la fase actual

La fase actual cubre documentacion y preparacion para implementar Learning. No incluye cambios funcionales.

Incluido en esta fase documental:

- Auditoria inicial del repo.
- PRD Learning.
- Roadmap por etapas.
- Alcance y fuera de alcance.
- Sistema operativo de contenido.
- Plan Figma-to-frontend.
- Plan tecnico propuesto.
- Registro de decisiones.

## Alcance del producto MVP

Incluido:

- Dashboard Learning.
- Aprendizaje de contenido medico.
- Repaso de temas.
- Practica con preguntas.
- Medicion de progreso.
- Contenido educativo revisable.

## Fuera de alcance del MVP

Clinical Care:

- Gestion de pacientes.
- Historia clinica.
- Triage.
- Diagnostico o apoyo a decision clinica.
- Tratamientos o recomendaciones para pacientes reales.
- Integracion con sistemas clinicos.

Teaching:

- Roles docentes.
- Gestion de cursos por profesor.
- Evaluaciones institucionales.
- Paneles de cohortes.
- Calificaciones o reportes academicos institucionales.

Otros fuera de alcance:

- Pagos.
- Marketplace.
- Comunidad o chat.
- Aplicacion movil nativa.
- Generacion automatica no revisada de contenido medico.
- Integraciones externas no necesarias para Learning MVP.

## Politicas de alcance

- Toda nueva pantalla debe mapearse a aprender, repasar, practicar o progreso.
- Toda nueva entidad de datos debe justificar su uso dentro de Learning.
- Toda referencia a Clinical Care o Teaching debe quedar como futuro potencial, no como requisito del MVP.
- El contenido medico debe presentarse como educativo, no como consejo clinico para pacientes.

## Senales de desvio de alcance

- Aparecen roles de medico tratante, paciente o docente antes de cerrar Learning.
- Se agregan flujos de diagnostico, tratamiento o seguimiento clinico.
- Se implementa administracion de cursos o calificaciones.
- Se agregan pantallas que no responden a una necesidad de estudio del estudiante.
- Se prioriza monetizacion antes de validar retencion y utilidad de Learning.
