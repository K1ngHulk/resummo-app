# Roadmap

## Principios

- Construir solo Learning hasta validar la experiencia base.
- Mantener Clinical Care y Teaching fuera del roadmap inicial de implementacion.
- Separar maqueta visual, contenido estructurado, practica y medicion de progreso.
- Evitar dependencias de backend hasta definir contratos de datos.

## MVP privado

Objetivo: validar la experiencia Learning con usuarios cercanos y contenido limitado.

Incluye:

- Dashboard Learning con navegacion basica.
- Datos mock o dataset local controlado.
- Flujo de continuar aprendizaje.
- Flujo simple de sesion de preguntas.
- Resumen de progreso basico.
- Primer mapa de componentes desde Figma.
- Checklist editorial para contenido medico.

No incluye:

- Registro publico.
- Pagos.
- Clinical Care.
- Teaching.
- Recomendaciones automaticas avanzadas.
- Analitica compleja.

Criterio de salida:

- Un estudiante puede entrar, entender que estudiar, practicar preguntas y ver un resumen de avance aunque los datos sean limitados.

## MVP publico

Objetivo: abrir una version limitada a mas estudiantes con base tecnica mas estable.

Incluye:

- Autenticacion si es necesaria para persistir progreso.
- Persistencia de progreso y sesiones.
- Biblioteca inicial de temas.
- Banco inicial de preguntas por tema.
- Estados de carga, error y vacio.
- Instrumentacion basica de eventos de aprendizaje.
- Politica de contenido y disclaimers educativos.

No incluye:

- Clinical Care.
- Teaching.
- Herramientas institucionales.
- Casos clinicos con datos reales de pacientes.

Criterio de salida:

- Usuarios externos pueden aprender y practicar con progreso persistente sin intervencion manual del equipo.

## V1

Objetivo: consolidar Learning como producto usable de estudio continuo.

Incluye:

- Rutas de estudio por tema o especialidad.
- Historial de sesiones.
- Explicaciones de respuestas.
- Filtros por tema, dificultad y estado.
- Panel de progreso mas detallado.
- Mejoras de accesibilidad y performance.
- Sistema editorial mas robusto para crear, revisar y publicar contenido.

No incluye salvo decision explicita posterior:

- Clinical Care.
- Teaching.

Criterio de salida:

- Resummo puede sostener un catalogo Learning creciente con medicion de progreso clara y mantenimiento editorial.

## V2

Objetivo: expandir Learning con personalizacion y capacidades avanzadas, sin asumir todavia otros dominios.

Posibles lineas:

- Recomendaciones de repaso segun desempeno.
- Planes de estudio personalizados.
- Practica adaptativa.
- Busqueda semantica sobre contenido validado.
- Analitica avanzada de debilidades.
- Importacion o gestion avanzada de contenido.

Pendiente de decision:

- Si Clinical Care o Teaching entran como productos separados despues de validar Learning.
- Si habra planes pagados, integraciones institucionales o roles de docente.
